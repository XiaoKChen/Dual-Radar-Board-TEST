### src/

- **`serial_logger.py`** — Production-quality binary frame capture tool: connects to the device serial port, sends `start [N]` / `stop` commands, aligns to the `RADR` magic header (struct: `<4sHHII` — magic, version, sample_size, frame_index, sample_count), reads binary payloads frame-by-frame, writes raw binary to an output file, and optionally produces a human-readable formatted dump organizing samples by chirp and RX antenna index. Supports continuous or N-frame-limited capture with a keyboard-interrupt stop path.

- **`format_binary_frames.py`** — Standalone binary frame decoder: reads `.bin` capture files produced by `serial_logger.py`, parses the same `RADR` header protocol, unpacks sample payloads (1/2/4-byte, signed or unsigned), and writes either a human-readable text dump (Frame → Chirp → Sample tree) or a CSV (`frame, chirp, sample, rx, value`). Accepts `--summary-only` for quick statistics without writing decoded output.

- **`radar_process.py`** — Full offline DSP pipeline script: parses a `frames.txt` log (or simulates targets synthetically), applies lowpass filtering, DC removal, and Hilbert transform to reconstruct the analytic signal, computes Range FFT (64-bin, ~0.33 m/bin) with peak detection, Doppler FFT (32-point, hanning-windowed) for velocity, and Digital Beamforming (DBF) for angle-of-arrival using steering vectors across RX antennas. Produces four matplotlib plots: range profile, Doppler spectrum, Range-Doppler map, and DBF beampattern.

- **`radar_process_test_001.py`** — Python re-implementation of the firmware presence detection algorithm (`presence_detection.c`) for offline parameter tuning: mirrors `presence_radar_settings.h` constants (61.02–61.48 GHz, 128 samples, 32 chirps, same 65-tap bandpass coefficients), implements macro-movement detection (FFT difference + range-intensity scaling) and micro-movement detection (per-bin Doppler FFT with mean removal) with configurable thresholds, intervals, and validity windows.

- **`radar_process_test.py`** — Extended multi-frame offline test harness: parses multi-frame `frames.txt` logs (Frame/Chirp/Sample hierarchy), implements Range-Doppler map generation with local peak detection (`detect_peaks` using `scipy.ndimage`), intended for batch validation across captured sequences.

- **`frames.txt`** — Sample captured radar frame data in human-readable text format (chirp/sample log); used as input by `radar_process.py` and `radar_process_test.py`.
