Owns all application-layer C firmware: the FreeRTOS entry point, dual-radar initialization and ISR wiring, the presence-detection DSP engine, and the USB-CDC logging subsystem.

- **Responsibility:** Application source — task creation, radar hardware bring-up, DSP algorithm, USB output
- **Key files:** `main.c`, `radar_task.c`, `presence_detection.c`, `presence_detection.h`, `presence_radar_settings.h`, `usb_logging.c`, `FreeRTOSConfig.h`
- **Relationships:** imports from `bsps/` (cybsp, cyhal, PDL), `deps/` (FreeRTOS, emUSB-Device, XENSIV sensor driver, CMSIS-DSP); exposes `RadarTask`, `init_dual_radars`, `get_radar_status`, `status_printf`, `presence_init`, `presence_process_frame`

---

- **`main.c`** — Firmware entry point: initializes BSP, UART retarget-io, and four FreeRTOS synchronization objects (`xRadar1Sem`, `xRadar2Sem`, `xSpiMutex`, `xStatusMutex`); spawns `SystemTask` which sequences USB init, dual-radar init, then creates `RadarTask` (priority 2) and `PrintTask` (priority 1) before deleting itself. `PrintTask` polls `get_radar_status()` every 3 s and emits presence state and range over USB-CDC.

- **`radar_task.h`** — Public interface for the radar subsystem: declares `radar_status_t` (holds `presence_state_t` + `float32_t range`), `RadarTask`, `init_dual_radars`, `get_radar_status`, and the four global FreeRTOS handles as `extern`.

- **`radar_task.c`** — Implements the dual-radar driver and FreeRTOS task. Defines ISR handlers `radar1_irq_handler` / `radar2_irq_handler` (post binary semaphore from ISR). `init_dual_radars` initializes a shared SPI bus at 12 MHz, hard-resets both BGT60TR13C sensors sequentially with individual CS lines, loads register config from `register_list`, wires interrupts, and calls `presence_init` for each context. `RadarTask` alternates between the two sensors each frame period: takes `xSpiMutex`, triggers frame acquisition, waits on the sensor semaphore (3 s timeout), reads FIFO, normalizes `uint16_t` samples to `float32_t` in `[0,1]`, averages across chirps, calls `presence_process_frame`, and updates the mutex-protected `radar_status_t` pair.

- **`presence_detection.h`** — Defines the entire DSP context and configuration. Key types: `presence_state_t` (ABSENCE / MACRO_PRESENCE / MICRO_PRESENCE), `presence_result_t` (state, `range_bin`, `range_m`, max values), `presence_context_t` (holds two ARM CMSIS-DSP FFT instances, Hamming window, range-intensity window, macro FFT double-buffer, FIR bandpass filter instances and states for up to 25 range bins, circular micro Doppler FFT buffer `[128 × 25]`, detection timestamps, and all state tracking variables). Declares `presence_init` and `presence_process_frame`.

- **`presence_detection.c`** — Implements the two-stage presence detection algorithm. `presence_init` sets up `arm_rfft_fast_instance_f32` (128-point real FFT), `arm_cfft_instance_f32` (128-point complex Doppler FFT), generates Hamming window, initializes per-range-bin 65-tap FIR bandpass filters (10–35 Hz coefficients generated in MATLAB), and calls `presence_reset`. `presence_process_frame` per call: (1) applies Hamming window and computes range FFT producing 64 complex bins; (2) optionally routes bins through per-bin FIR bandpass filters; (3) macro detection: every 250 ms compares current vs. previous FFT magnitude deltas scaled by a range-intensity ramp, timestamps hits with 1 s validity; (4) stores range-bin columns into a circular 128-row Doppler buffer; (5) micro detection: applies mean removal and 128-point complex Doppler FFT to each range-bin column, sums low-bin magnitudes (bins 1–5) as a micro-motion score, timestamps hits with 4 s validity; (6) resolves final `presence_result_t` from timestamp expiry logic with ABSENCE fallback.

- **`presence_radar_settings.h`** — XENSIV BGT60TR13C register configuration header (generated by SDK configurator v3.6.4): defines `XENSIV_BGT60TRXX_CONF_*` constants (61.02–61.48 GHz sweep, 128 samples/chirp, 32 chirps/frame, 1 RX antenna, ~50 ms frame period, 38 registers) and the literal `register_list[]` array (guarded by `XENSIV_BGT60TRXX_CONF_IMPL`).

- **`usb_logging.h`** — Public interface for the USB-CDC logging subsystem: declares `init_usb_logging`, `wait_for_usb_configured`, and `status_printf`.

- **`usb_logging.c`** — Implements the emUSB-Device CDC channel. `init_usb_logging` calls `USBD_Init`, configures bulk IN/OUT and interrupt IN endpoints, registers the CDC class, sets device info (VID `0x058B`, PID `0x027D`, "CDC Radar Data"), and calls `USBD_Start`. `wait_for_usb_configured` polls `USBD_GetState` with a 10 s timeout. `status_printf` formats into a 256-byte stack buffer via `vsnprintf` and non-blocking-writes to the CDC handle if the device is enumerated.

- **`FreeRTOSConfig.h`** — FreeRTOS kernel configuration for PSoC 6 CM4: 1 kHz tick, 7 priority levels, heap_4 with 48 KB total heap, stack overflow check mode 2, `malloc` failure hook enabled, software timers enabled, CMSIS interrupt vector remapping (`SVC_Handler`, `PendSV_Handler`, `SysTick_Handler`), tickless idle disabled.
