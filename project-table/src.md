- **main.c** — Application entry point. `main()` calls `cybsp_init()`, enables IRQs, initializes retarget-io UART (`P7_1`/`P7_0`), creates the four synchronization objects (`xRadar1Sem`, `xRadar2Sem`, `xSpiMutex`, `xStatusMutex`), spawns `SystemTask` (priority 3), and starts the FreeRTOS scheduler. `SystemTask` inits the LED (`P10_3`), calls the no-op USB stubs, calls `init_dual_radars()`, then spawns `RadarTask` (priority 2) and `PrintTask` (priority 1) before deleting itself — three tasks total. `fill_record()` converts a `radar_status_t` into a wire-ready `radar_sensor_record_t`, clamping distance to 0–65534 mm and angle to ±32767 centidegrees, using sentinels (`RADAR_DISTANCE_MM_NONE`/`RADAR_ANGLE_CDEG_NONE`) when state is `PRESENCE_STATE_ABSENCE`. `PrintTask` loops every 3000 ms, calls `get_radar_status`, fills two records, and calls `usb_log_presence_binary`.
- **radar_task.c / radar_task.h** — Dual-radar SPI driver and orchestration. Defines the shared `xensiv_bgt60trxx_mtb_t sensor1`/`sensor2`, `radar_status_t` struct (state, range, distance_m, angle_deg, range_bin, doppler_bin), and the four extern semaphore/mutex handles. `init_dual_radars()` configures a shared 12 MHz SPI bus (`XENSIV_BGT60TRXX_SPI_FREQUENCY`), performs a hardware reset sequence, initializes each sensor with the 38-entry `register_list[]`, and calls `presence_init()` for each context. `radar1_irq_handler`/`radar2_irq_handler` are GPIO-event ISRs that give the per-sensor binary semaphore. `run_presence_detection()` triggers a frame, blocks (3 s timeout) on the semaphore, reads the FIFO via `xensiv_bgt60trxx_get_fifo_data`, normalizes ADC samples (`/4096.0f`), de-interleaves and chirp-averages RX2/RX3, calls `presence_process_frame` on RX2, then computes azimuth from the RX2/RX3 phase difference at the peak range bin (`arg(RX3·conj(RX2))`, `RADAR_RX_SPACING_LAMBDA=0.5`) and clamps it via `asinf`. `RadarTask` is the FreeRTOS task looping at `XENSIV_BGT60TRXX_CONF_FRAME_REPETITION_TIME_S` (≈150 ms), servicing both sensors under `xSpiMutex` each cycle, toggling `LED1` as an activity indicator.
- **presence_detection.c / presence_detection.h** — Core CMSIS-DSP signal-processing pipeline, independent of hardware I/O. Header defines `PRESENCE_NUM_SAMPLES=128`, `PRESENCE_NUM_CHIRPS=32`, `PRESENCE_MACRO_FFT_SIZE=64`, `PRESENCE_MICRO_FFT_SIZE=128`, `PRESENCE_MIN_RANGE_BIN=2`, `PRESENCE_MAX_RANGE_BIN=56` (~6.0 m gate at current ~0.107 m/bin resolution), thresholds (`PRESENCE_MACRO_THRESHOLD=1.0`, `PRESENCE_MICRO_THRESHOLD=25.0`), timing constants (250 ms macro-compare interval, 1000 ms macro validity, 4000 ms micro validity), and the `presence_context_t`/`presence_result_t`/`presence_state_t` types. `PRESENCE_BANDPASS_ENABLED` is `0` — the 65-tap FIR bandpass code (coefficients from `fir1(64,[10/100 35/100],'DC-1')`) is present but compiled out, with a comment noting it would overflow RAM at the current `PRESENCE_MAX_RANGE_BIN`. `presence_init()` builds the Hamming window and range-intensity window (`0.2*(i+1)`) and resets state. `presence_process_frame()` does: (1) Hamming-windowed 128-point real FFT via `arm_rfft_fast_f32` into 64 complex bins; (2) optional bandpass (disabled); (3) macro detection by diffing against a 250 ms-old snapshot per bin 2–56, scaled by the range-intensity window, tracking a hit counter (`PRESENCE_MACRO_CONFIRMATIONS=0`) and per-bin validity timestamps; (4) writes the current FFT row into a 128-row circular micro buffer; (5) once full, extracts one range-bin column per call, removes its mean, runs a 128-point complex FFT (`arm_cfft_f32`), sums magnitude of Doppler bins 1–5 as a "speed" score against `PRESENCE_MICRO_THRESHOLD=25.0`, and on a hit records a signed peak-Doppler-bin (converting bins ≥64 to negative); (6) resolves overall state transitions (ABSENCE → MACRO_PRESENCE → MICRO_PRESENCE) based on which per-bin timestamps are still valid.
