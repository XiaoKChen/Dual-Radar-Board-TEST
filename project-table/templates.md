Stores frozen `design.modus` peripheral configurations for the full set of supported PSoC 6 evaluation boards, plus a `TEST-000/` directory containing a prior single-radar binary-streaming firmware snapshot.

- **Responsibility:** Reference configurations — per-target device configurator files and a historical firmware baseline
- **Key files:** `TEST-000/main.c`, `TEST-000/presence_radar_settings.h`, `TARGET_CY8CPROTO-062S3-4343W/config/design.modus` (and equivalents for 13 other targets)
- **Relationships:** not compiled into the active build; serves as development history and multi-target porting reference

---

### TEST-000/

- **`main.c`** — Earlier single-radar binary streaming firmware: initializes one BGT60TR13C over SPI, handles UART commands (`start [N]` / `stop`), transmits raw ADC samples wrapped in a 16-byte binary frame header (`RADR` magic, version=1, `uint16_t` sample_size, frame_index, sample_count) over UART; this is the firmware the `serial_logger.py` capture protocol was designed against.

- **`presence_radar_settings.h`** — Alternate radar register configuration: 60.999–62.2 GHz sweep (wider bandwidth), 64 chirps/frame, 3 RX antennas — a different operating mode from the active dual-radar config in `src/`.

### TARGET_*/config/

- **`design.modus`** — Per-board ModusToolbox Device Configurator project files for 13 alternative kit targets (CY8CEVAL-062S2 variants, CY8CKIT-062 variants, CY8CPROTO-062 variants, CYW9P62S1 variants); provide peripheral and pin configurations for porting the example to those boards.
